Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: Grid search techniques is used to search for best set of parameters of the learning model. It is basically an exhaustive search for the best set of parameters that will yield the best prediction performance for the learning model. Every single parameter has potential values. Grid search picks one set of parameter-value and trains the model and evaluates its performance. It does this for every single combination of potential parameter assignments. At the end the assignment that yields the best prediction accuracy is chosen as the sub-optimal parameter setting.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: k-fold cross-validation increases the reliability of the trained model by exposing the model to various training and testing situations.

(i)Data is divided into k equal-sized subsets with enough shuffling to avoid biased grouping. (ii)The model is trained with k-1 chunks and tested on the kth chunk. This repeats till k many training/testing session is done. (iii) Validation results are averaged at the end. (iv) The model which gives the best result with this technique is picked.

If normal hold-method is used to train the set on certain part of the dataset and test it with the remaning part, the learning algorithm will only fit to the one set of data. That means it will be more prone to the problems that might occur due to missing points, outliers, or basically it will be biased towards how the set is divided. k-fold CV avoids all those for a given dataset by exposing the learning model to k different learning and testing situations.

For Grid search, k-fold cross validation means a richer sampling. Therefore, k-fold CV helps choosing the more optimal set of parameter assignments. If we don't use cross-validated set, then grid search would potentially end up with more locally optimal set of parameter asssignments.
